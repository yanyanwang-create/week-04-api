Week 5 Reflection
1. What is the difference between the system prompt and the user message? Why does the separation matter?

The system prompt defines the assistant’s role, behavior, and rules. The user message contains the specific request from the user. The separation matters because it helps keep the AI focused on its intended purpose while still allowing users to ask different questions.

2. What happened when you changed the system prompt in Part 4? What surprised you most?

Changing the system prompt changed the style and structure of the responses. A more detailed prompt produced more organized recommendations, while a stricter prompt kept the assistant focused only on books. I was surprised that small changes in the prompt could significantly affect the output.

3. Name one situation where using AI in an app could cause harm, and how would you mitigate it?

AI could provide inaccurate recommendations or misleading information. To reduce this risk, users should be informed that AI responses are suggestions, and important decisions should not rely entirely on AI-generated content.

4. If you had infinite Claude API credits, what AI feature would you add to this book tracker? Describe it technically?

I would add an AI reading assistant that automatically summarizes books, tracks reading progress, and generates personalized recommendations based on reading history. The backend would retrieve book data from the database, send the information to Claude, and return personalized reading plans and recommendations to the frontend.
